Configuración de Ubiquitous y Google Cloud

Fecha: 20 de Enero, 2026

1. Objetivo y Contexto

Objetivo: Configurar y desplegar entornos de trabajo remotos mediante el uso de infraestructura universitaria (Ubiquitous) y servicios de computación en la nube (Google Cloud Platform). El propósito es establecer canales seguros de comunicación (SSH/SFTP) y aprovisionar una instancia de servidor Linux para el despliegue de bases de datos.

Contexto: En el desarrollo profesional de software, el trabajo rara vez se realiza en la máquina local ("localhost") de manera aislada. Se requiere el dominio de servidores remotos para el despliegue de aplicaciones. Utilizamos las cuentas institucionales para simular un entorno de producción real, gestionando credenciales de acceso, configuración de puertos y sistemas operativos basados en Unix.

2. Conceptos y Profundización

  • IaaS (Infrastructure as a Service): Modelo de servicio en la nube donde el proveedor (Google Cloud) ofrece recursos de computación virtualizados a través de internet. A diferencia del PaaS o SaaS, aquí gestionamos el sistema operativo, lo que nos da control total sobre la instalación de software como MariaDB.
  • Protocolo SSH (Secure Shell): Estándar criptográfico de red que permite operar servicios de red de forma segura sobre una red insegura. Utiliza una arquitectura cliente-servidor y, típicamente, el puerto 22. Es fundamental para la administración remota de sistemas Linux sin interfaz gráfica (headless).
  • SFTP (SSH File Transfer Protocol): A diferencia del FTP tradicional que envía datos en texto plano, SFTP utiliza el túnel seguro de SSH para la transferencia de archivos. Fue utilizado mediante CyberDuck para desplegar el portafolio web en el servidor Ubiquitous.
  • Virtual Machine (VM): Emulación de un sistema informático. Las instancias de Compute Engine en GCP actúan como servidores dedicados virtuales, permitiendo aislamiento, escalabilidad y la capacidad de "levantar" o "destruir" entornos de desarrollo en segundos.

5. Conclusiones

Aprendizaje: La competencia adquirida más relevante es la capacidad de administrar servidores sin depender de una interfaz gráfica, utilizando la terminal para la gestión de paquetería y servicios. Entendí la diferencia crítica entre un servidor de archivos (Ubiquitous) y un servidor de procesamiento/base de datos (GCP).

Problemas y Soluciones: Hubo complicaciones con las reglas de firewall en GCP que impedían la conexión inicial; se solucionó configurando correctamente las reglas de VPC para permitir tráfico SSH. También fue necesario regenerar las llaves SSH locales para evitar conflictos de autenticación.

Mejoras Futuras: Implementar llaves SSH públicas/privadas para autenticación sin contraseña y configurar un alias en el archivo `.ssh/config` para agilizar el acceso a las máquinas virtuales.
