Descarga de archivos y edición en terminal

Fecha: 3 de Febrero, 2026

1. Objetivo y Contexto

Objetivo: Automatizar la ingesta de scripts SQL externos utilizando herramientas de línea de comandos (`wget`) y realizar modificaciones "in-situ" en el servidor utilizando editores de texto modales (`vi`). Posteriormente, ejecutar la importación de estos volcados (dumps) tanto en MariaDB como en PostgreSQL para verificar la portabilidad de los datos.

Contexto: En entornos de servidores de producción, rara vez se cuenta con navegadores web para descargar archivos. La capacidad de obtener recursos directamente desde la terminal y editarlos sin depender de una GUI local es una habilidad fundamental para cualquier administrador de bases de datos (DBA).

2. Conceptos y Profundización

  • Wget (World Wide Web Getter): Herramienta no interactiva de red para descargar archivos vía HTTP, HTTPS y FTP. Su robustez permite reanudar descargas interrumpidas y operar en segundo plano, esencial para scripts de automatización.
  • Editor VI/VIM: Editor de texto estándar en sistemas Unix. Funciona mediante "modos": el Modo Comando (para navegar, copiar, guardar) y el Modo Inserción (para escribir). Su eficiencia radica en que no requiere levantar manos del teclado.
  • Redirección de Entrada/Salida: El uso de operadores como `>` (redirigir salida a archivo) y `<` (redirigir archivo a entrada de comando) es la base para importar/exportar bases de datos. Ejemplo: `mysql ... < archivo.sql` inyecta el contenido del archivo al proceso del cliente MySQL.
  • Portabilidad SQL: Los volcados SQL a menudo requieren ajustes menores al moverse entre DBMS (ej. comillas, tipos de datos específicos) que deben corregirse manualmente con VI.

4. Entregables

Comandos utilizados:

# Descarga
$ wget https://ubiquitous.udem.edu/~raulms/content/SC3303/Biblioteca.sql

# Importación en MariaDB
$ mysql -u biblio_user -p Biblioteca < Biblioteca.sql

# Respaldo lógico solo datos
$ mysqldump -u biblio_user -p --no-create-info Biblioteca > datos.sql

# Operaciones en PostgreSQL
$ psql -U postgres -f Biblioteca.sql
$ pg_dumpall -U postgres -f respaldo_pg.sql

5. Conclusiones

Aprendizaje: Dominar `vi` es esencial; al principio es anti-intuitivo por los modos, pero permite ediciones rápidas de archivos SQL gigantes que bloquearían un editor gráfico tradicional. Entendí cómo los operadores de redirección (`<`, `>`) conectan el sistema de archivos con los procesos del DBMS.

Problemas y Soluciones: El archivo SQL original venía con sintaxis específica de MySQL que fallaba en Postgres. Usé `vi` para buscar y reemplazar los tipos de datos incompatibles directamente en el servidor.

Mejoras Futuras: Aprender comandos avanzados de `sed` o `awk` para automatizar el reemplazo de sintaxis incompatible sin tener que abrir el archivo manualmente.
